# EduStorm 抖音数据同步总览

这是第一版内部“内容矩阵作战台”，目标不是自动回复，而是把抖音矩阵的数据稳定汇总到一个总览里。

## 当前已实现

- `/matrix`：抖音数据同步总览页面。
- `/api/matrix/overview`：总览数据接口。
- SQLite 数据表：
  - `matrix_accounts`：抖音账号与同步状态。
  - `matrix_works`：作品母版。
  - `matrix_snapshots`：每日数据快照。
  - `matrix_keywords`：私信/评论关键词。
  - `matrix_sync_logs`：同步日志。

当前已接入抖音创作者中心只读同步器：

```bash
npm run matrix:sync-douyin
```

同步器会复用 `.playwright/douyin-profile` 登录态，读取作品管理接口，并写入：

- 作品标题
- 发布时间
- 封面 URL
- 视频链接
- 播放量
- 点赞数
- 评论数
- 分享数

私信数、微信进量、有效线索暂未接入，仍保持为人工或第三方系统后续导入字段。

## 本地启动

```bash
npm run matrix
```

打开：

```text

```

接口：

```text

```

## 数据源策略

总览页不绑定某一种采集方式。推荐按优先级接入：

1. 官方平台接口
   - 如果抖音/巨量/创作者服务提供作品数据、账号数据、评论或私信相关接口，优先使用。
   - 官方接口稳定性和合规性最好，但通常需要资质、授权、审核或特定账号权限。

2. 创作者中心只读采集
   - 用 Playwright 复用本机登录态。
   - 只读取作品数据、评论数据、私信会话和账号状态。
   - 不自动回复、不主动私信、不批量操作。
   - 当前抖音作品同步使用 `creator.douyin.com/janus/douyin/creator/pc/work_list` 页面接口读取作品数据。

3. 导出文件导入
   - 如果平台后台能导出 Excel/CSV，就先让运营下载到固定目录。
   - 脚本清洗后写入标准表。

## 标准数据流

```text
数据源
  -> 采集适配器
  -> 标准化字段
  -> SQLite 快照表
  -> /api/matrix/overview
  -> /matrix 总览
```

## 后续真实采集器要写入的字段

作品基础信息：

```text
作品ID
作品标题
账号ID
封面
封面URL
封面本地缓存路径
发布时间
视频链接
内容标签
状态标签
```

封面是作品墙的第一判断信息，真实同步器必须采集。建议同时保存两份：

- `cover_url`：平台页面或官方接口返回的封面地址。
- `cover_local_path`：下载到本地 `data/covers/` 的缓存图，避免平台链接过期或带鉴权参数失效。

每日快照：

```text
作品ID
采集时间
播放量
点赞数
评论数
收藏数
转发数
私信数
微信进量
有效线索
平均观看时长
完播率
封面点击率
```

账号状态：

```text
账号ID
账号名称
在线状态
同步状态
最近同步时间
负责人
```

## 安全边界

第一阶段只做数据读取：

- 不自动回复。
- 不主动私信。
- 不批量关注/点赞/评论。
- 不绕验证码。
- 不高频请求。
- 不把 Cookie 明文暴露到页面。

核心目标是让团队每天看到：

```text
哪条作品有流量
哪条作品有私信
哪条作品带来微信进量
哪条作品值得复盘和二创
哪个账号需要重新登录
```
